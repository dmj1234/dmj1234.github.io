发表于 2021-01-14 更新于 2021-01-13

这里说下关于报错的问题

  • 我在npm run docs:build时报错样式：

  • 我从toast.vue这个组件中找错，我看了一遍代码同时WebStorm没有报错，我想这个文件导致的话，我就删了。结果删了之后后面一个组件也报了一样的错。所以文件可能出错！！！

  • 然后我往下看错误，This may cause things to work incorrectly. Make sure to use the same version for both.他说vue的版本不一样，我将依vue的去配置去配置 “vue-template-compiler”: “^2.6.12”,然后问题依然是一样的，然后他说- vue@2.6.12和vue-server-renderer@2.5.16不一致，我配置成一样的结果还是错了。

  • 在这之前我把node_modules删了，网上说有之前的缓存。我重新配置了结果依然报错，错误一样。

  • 这个问题解决后真的太搞笑了。比如说 “vue-server-renderer”: “2.6.12”,/ “vue”: “2.6.12”,/ “vue-template-compiler”: “^2.6.12”,这三个是兄弟，我按照最大或的版本分配这样就不公平，我就按中间的设置成一样的，问题就是这样解决了。

  • 关于node_modules的报错问题，如果还成功我就直接去对应文件找上门了。
